# tasks/README.md

这里是总台的任务看板目录。

## 用法

每个正式任务创建一个单独 markdown 文件，命名建议：

`YYYY-MM-DD-任务短名.md`

例如：

`2026-03-21-login-redesign.md`

## 每个任务至少包含

- 任务名
- 目标
- 优先级
- 参与角色
- 当前状态
- 子任务列表
- 风险/阻塞
- 最近更新
- 下一步

## 状态建议

- Backlog
- Scoped
- Designed
- In Progress
- In Review
- QA Passed
- Ready to Deploy
- Deployed
- Blocked
- Done

## 原则

- 一个任务一个文件，方便持续追踪
- 总台负责维护状态变化
- 子 agent 产出可以汇总贴到对应任务文件里
- 给用户汇报时，以任务文件中的最新结论为准
