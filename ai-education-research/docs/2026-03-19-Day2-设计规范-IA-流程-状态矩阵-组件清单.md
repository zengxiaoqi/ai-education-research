- 日期：2026-03-19
- 目标：把“误区诊断引擎（A）+ 教师运营台（C）”落地成可开发的 Web MVP 规范

## 0. 产品边界（Day2 对齐）

### 0.1 MVP 目标
在 **2 周内上线可试用 Web MVP**，完成以下闭环：
1. 教师创建班级与作业任务
2. 学生提交“答案 + 解题过程”
3. 系统输出误区类型 + 纠偏建议
4. 教师在看板查看班级薄弱点与复核队列

### 0.2 目标用户
- 主用户：中学/高校理工科教师（班级管理 + 反馈）
- 次用户：学生（做题 + 接收纠偏反馈）

### 0.3 非目标（本期不做）
- 家长端独立应用
- 完整 LMS 对接
- 自动生成高风险成绩单（只给建议，不自动定分）

## 1) 信息架构（Information Architecture）

## 1.1 页面树（Page Tree）

- `/` 登录页
- `/onboarding`
  - `/onboarding/school-profile`（学校/学科/年级配置）
- `/teacher`
  - `/teacher/dashboard`（教师首页）
  - `/teacher/classes`（班级列表）
  - `/teacher/classes/:classId`（班级详情）
  - `/teacher/assignments`（作业列表）
  - `/teacher/assignments/new`（创建作业）
  - `/teacher/assignments/:assignmentId`（作业详情）
  - `/teacher/review-queue`（低置信复核队列）
  - `/teacher/insights`（班级薄弱点洞察）
  - `/teacher/settings`（评分策略、隐私设置）
- `/student`
  - `/student/home`（学生首页）
  - `/student/assignments/:assignmentId`（作答页）
  - `/student/results/:submissionId`（反馈结果页）
- `/shared`
  - `/shared/not-found`
  - `/shared/error`

## 1.2 导航模型（Navigation Model）

### 教师端
- 左侧主导航：Dashboard / Classes / Assignments / Review Queue / Insights / Settings
- 顶部全局：班级切换、全局搜索、通知、用户菜单

### 学生端
- 顶部简化导航：Home / 当前任务 / 历史反馈
- 作答页沉浸模式（隐藏复杂导航）

## 1.3 入口点（Entry Points）
- 教师：登录后默认进入 Dashboard
- 学生：通过班级码/任务链接直接进入作答页
- 复核入口：当评分置信度 < 阈值时自动进入 Review Queue

## 2) 核心流程（Core Flows）

## 2.1 Happy Path（教师）
1. 教师创建班级
2. 创建作业（题目 + 评分标准 + 截止时间）
3. 发布作业，学生开始提交
4. 系统完成误区诊断与建议生成
5. 教师在 Dashboard 看到班级概览，在 Insights 查看误区分布
6. 教师处理低置信复核项并发布反馈

## 2.2 Happy Path（学生）
1. 打开作业
2. 输入答案 + 解题过程
3. 提交后查看反馈：
   - 误区类型
   - 错因解释
   - 1~3 道纠偏练习题
4. 完成纠偏题，系统更新“二次正确率”

## 2.3 Error / Edge Paths
- 截止后提交：提示不可提交，可申请延时
- 网络中断：自动草稿保存，恢复后继续
- 内容过短：触发“请补充解题过程”校验
- 模型服务超时：返回兜底提示 + 重试按钮

## 2.4 Permission / Empty Paths
- 教师账号无班级：展示“创建第一个班级”引导
- 学生无任务：展示空态和加入班级入口
- Insights 无数据：提示“需至少 N 份提交后生成洞察”

## 3) 页面规格（Screen Specs）

## 3.1 教师 Dashboard
- 目标：快速掌握班级学习健康度
- 核心组件：
  - KPI 卡片（提交率、平均置信度、二次正确率）
  - 误区Top5条形图
  - 待复核队列预览
- 主操作：进入复核队列 / 发布班级反馈
- 校验：时间范围选择必须有效

## 3.2 创建作业页
- 目标：低成本配置任务
- 核心组件：
  - 题目编辑器
  - 评分规则模板（严谨/标准/宽松）
  - 误区标签预设（可多选）
- 主操作：发布作业
- 校验：题目、截止时间、评分规则必填

## 3.3 学生作答页
- 目标：收集可诊断输入
- 核心组件：
  - 答案输入区
  - 解题过程输入区（引导式提示）
  - 草稿自动保存
- 主操作：提交作答
- 校验：答案与过程至少其一达到最小字数；敏感词过滤

## 3.4 反馈结果页
- 目标：帮助学生纠偏，不止给结论
- 核心组件：
  - 误区诊断卡（类型 + 证据句）
  - 纠偏建议卡（下一步策略）
  - 练习推荐区（1~3题）
- 主操作：开始纠偏练习
- 校验：若模型置信度过低，提示“教师复核中”

## 3.5 复核队列页
- 目标：教师高效处理低置信结果
- 核心组件：
  - 队列表格（学生、题目、置信度、提交时间）
  - 对比视图（原答案 vs 模型判断）
  - 一键修订并发布
- 主操作：确认/修订/驳回
- 校验：发布前必须选择处理动作

## 4) 组件规则（Component Rules）

## 4.1 按钮
- 尺寸：S(28) / M(36) / L(44)
- 类型：Primary / Secondary / Ghost / Danger
- 状态：default / hover / active / loading / disabled
- 规则：同一区域最多 1 个 Primary

## 4.2 表单
- 标签永远可见，不用 placeholder 替代 label
- 实时校验 + 提交校验双层机制
- 错误提示就近显示（字段下方）
- 辅助文本用于填写建议与示例

## 4.3 列表/表格
- 默认支持排序、筛选、分页
- 行级快捷操作：查看详情 / 复核 / 导出
- 空态必须提供下一步 CTA

## 4.4 反馈组件
- Toast：轻提示（成功/信息）
- Inline Error：字段级错误
- Modal：高风险动作（删除作业、批量发布）

## 5) 状态矩阵（State Matrix）

| 页面/模块 | Loading | Empty | Success | Error | Disabled |
|---|---|---|---|---|---|
| 教师 Dashboard | 骨架屏 + KPI 占位 | 无班级数据提示 + 创建按钮 | KPI+图表完整展示 | 拉取失败 + 重试 | 时间筛选不可用时禁用导出 |
| 创建作业 | 编辑器骨架 | 默认模板引导 | 作业发布成功提示 | 必填缺失/保存失败 | 发布中按钮禁用 |
| 学生作答 | 表单骨架 + 自动恢复草稿 | 无任务提示 | 提交成功跳转结果页 | 提交失败 + 本地草稿保留 | 截止后禁提 |
| 反馈结果 | 反馈卡占位 | 复核中提示 | 误区+建议+练习完整 | 诊断失败提示并允许重试 | 练习未生成时按钮禁用 |
| 复核队列 | 表格骨架 | 无待复核项 | 队列与详情联动 | 拉取失败/提交失败 | 无权限用户禁操作 |

## 6) 响应式规则（Responsive Rules）

## 6.1 Breakpoints
- Mobile: < 768
- Tablet: 768 - 1199
- Desktop: >= 1200

## 6.2 布局行为
- Desktop：双栏（导航 + 内容）
- Tablet：可折叠侧栏
- Mobile：底部导航 + 单栏流程

## 6.3 信息折叠优先级
1. 保留核心任务 CTA
2. 保留关键指标（提交率、误区Top3）
3. 次级图表折叠到“查看更多”

## 8) 无障碍（Accessibility）

- 键盘可达：所有主操作支持 Tab + Enter
- 对比度：文本与背景 >= WCAG AA
- Focus Ring：统一蓝色外描边（2px）
- 表单：全部字段有 label 与错误关联说明
- 图表：必须提供文本摘要（供读屏）
