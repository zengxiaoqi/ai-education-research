### 关键架构特点
1. **Gateway 是核心服务**：HTTP + WebSocket 双协议，处理所有 AI 对话
2. **Agent 系统庞大**：支持 Pi（嵌入式 agent）、ACP（agent 通信协议）、子 agent 调度
3. **多 LLM 支持**：通过 extensions 接入 OpenAI、Anthropic、Google、DeepSeek 等 50+ 提供商
4. **多渠道接入**：Discord、Telegram、WhatsApp、飞书、Signal、Slack 等 20+ 渠道
5. **插件化架构**：extension 是独立的 NPM 包风格，通过 plugin-sdk 接入
6. **技能系统**：skills/ 下是独立的技能包，可安装和管理
7. **移动端支持**：有 Android、iOS、macOS 原生应用

### extensions 目录（按类别）
- **LLM 提供商**：openai, anthropic, google, deepseek, ollama, groq, mistral, qwen 等
- **渠道**：discord, telegram, whatsapp, feishu, signal, slack, irc, line, matrix 等
- **能力扩展**：browser, memory-core, diffs, voice-call, image-generation-core 等

### 入口文件
- `openclaw.mjs` → `src/entry.ts`（CLI 入口）
- `src/index.ts`（库入口，导出 library 函数）

## 总体架构分析

派了 5 个 agent 并行分析各核心模块：
- ✅ arch-agents（Agent 引擎）
- ✅ arch-gateway（Gateway 服务）
- ✅ arch-plugins（插件/扩展系统）
- ✅ arch-infra（基础设施层）
- ✅ arch-channels（消息渠道系统）

完整分析报告已保存到 `memory/openclaw-architecture-analysis.md`

### 架构要点：
1. Gateway 是核心枢纽：HTTP+WS 双协议，承载所有请求
2. Agent 引擎是最大模块（1170文件）：Pi 嵌入式执行 + 多 LLM transport + 工具管道 + 子 agent 调度
3. 插件系统是声明式的：definePluginEntry → register(api) → hooks 注入
4. 消息渠道通过 extensions 实现：auto-reply/dispatch 路由 → gateway → agent
5. 配置支持热更新，沙箱支持 Docker/SSH 两种后端
